Adding a New Trigger Rule Using the Add Wizard

After a new trigger rule is added, you can activate or deactivate it for the running application, as described in "Activating and Deactivating a Rule". To create a trigger rule, you must at least complete Step 1 though Step 3 in the following procedure.

To add a new trigger rule by using the Add wizard:

  1. Click the Triggers tab.

  2. Click Add.

    The Add New Rule—Select attribute subscription trigger dialog box appears.

  3. Drill down to locate the attribute on which you want to trigger.

  4. Click Next.

    The Add New Trigger Rule—Set condition for triggering dialog box appears.

    In this dialog box, set the properties that define when the rule will be triggered; for example, the threshold at which the rule is invoked and for how long the triggering condition should be sustained before invoking the trigger action.

  5. Fill in the fields as described in Table: Set Condition for Triggering Field Descriptions.

    Set Condition for Triggering Field Descriptions

    In this field Enter

    Max trigger value

    The maximum value for the selected attribute at which point the rule should be triggered.

    Sustained (s)

    The duration, in seconds (s), that the triggering condition must be met for the action to be triggered.

    Limit period (s)

    The minimum time, in seconds (s), between consecutive triggers.

    Trigger on descending flank (checkbox)

    Whether or not you want the action to be triggered when the attribute reaches the trigger value from a higher value; for example, if the trigger value is 80 and the attribute value descends from 100 to 80, the notification will be triggered.

    Trigger on ascending flank (checkbox)

    Whether or not you want the action to be triggered when the attribute reaches the trigger value from a lower value; for example, if the trigger value is 80 and th attribute value ascends from 60 to 80, the notification will be triggered.

  6. Click Next.

    The Add New Trigger Rule—Select Action dialog box appears.

    Use this dialog box to identify the type of action you want to occur when a notification is sent. You have a choice of five options:

    • Send an alert to the application.

    • Send a warning to standard output.

    • Dump a flight recording.

    • Invoke a diagnostic command.

    • Send an e-mail to a specific recipient.

    • Start a JRA recording.

    • Create a thread stack dump.

  7. Select the action you want performed. Depending on the action you select, you might have to enter additional data, as described in Table: Selecting a Notification Action.

    Selecting a Notification Action

    Select To

    Application alert

    Print a notification in the Trigger Alerts dialog box. See"Viewing Application Alerts" for an example of this kind of action.

    Console output

    Print a notification on the console (system.out).

    Dump Flight Recording

    Create afile showing system activity leading up to the alert.

    Invoke Diagnostic Command

    Run a diagnostic command on the JVM.

    Start JRA Recording

    Starts a JRockit Runtime Analyzer reccording on the VM that triggered the rule. This option only works with Oracle JRockit versions prior to R28.0.0.

    Send e-mail

    Send an e-mail to a specified user. If you select this option, you must enter the following data in the fields that appear:

    • To (the e-mail recipient).

    • CC (anyone—aside from the recipient—to whom you want to send a copy of the e-mail).

    • BCC (anyone—aside from the recipient—to whom you want to send a blind copy of the e-mail)

    Thread stack dump

    Create a thread stack dump and log it to a file or display it as an application alert. If you select this option, you must enter the following data in the fields that appear:

    • The output method (system alert or log to file).

    • If you select log to file, the name of the file to which you want the dump written.

  8. Click Next.

    The Add New Trigger Rule—Select constraints dialog box appears.

    This is an optional step that allows you to limit when an action is triggered; for example, you might want your action to trigger only every other day of the week or during specific hours. If you do not want to set a constraint, skip Step 9 and go on to Step 10.

  9. Select the constraints you want applied to the trigger rule.

    • Date span sets a range of dates.

    • Day of week sets which days of the week you want your action to trigger. If you select this option, you must also specify which days you want your action to trigger.

    • Time span sets the time period.

  10. Click Next.

    The Add New Trigger Rule—Name your rule dialog box appears.

  11. Enter a name for your new rule and click Finish.

    The new rule appears on the Trigger Rules list.

    Your next step is to activate the rule, as described in "Activating and Deactivating a Rule".
