Analyzing Live Set Trends

From the Trend tab, start analyzing your application. The types with the highest growth in bytes per second are marked red (darkest) in the Trend Analysis table and they are listed at the top of the table. For each update, the list might change according to the current object allocation and garbage collection rates.

Table: Trend Tab Description describes the Trend tab columns.

Trend Tab Description

Column Displays

Type

The class name. The icon indicates if this is an array type and if it has multiple Class Loaders..

Type ID

A numerical identifier for the type.

Class Loader

The name of the associated class loader instance.

Class Loader ID

An alpha-numeric identifier for the class loader

Growth Rate

The increase in amount of memory (in bytes) used by all objects of this type, per second.

% of Heap

How much of the Java heap is occupied by objects of this type, measured in percentages of the entire heap.

Count

The number of objects of this type.

Delta Count

The difference between the current count and the count at the time you reset the delta.

Size

The amount of memory (in bytes) used by all objects of this type.

Delta Size

The difference between the current size and the size at the time you reset the delta.
