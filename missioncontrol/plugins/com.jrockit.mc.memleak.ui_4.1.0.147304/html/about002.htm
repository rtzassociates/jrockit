Extension Point for Instance Visualization

In earlier versions of the Memory Leak Detector, instances were normally shown as just the class name and a numerical ID. Sometimes the class name and ID of the class loader (which also is an instance) was appended. To make instances easier to identify, more class specific description of instances, similar to what the toString() method often provides are now used on the Instance List and the Instance relations tab.
